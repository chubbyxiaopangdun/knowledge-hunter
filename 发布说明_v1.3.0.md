## 🎯 核心更新：多知识库支持

从「单一飞书」升级为「飞书 + Obsidian 双知识库」

### 解决的问题

- 用户可能同时使用多个知识库管理内容
- Obsidian 用户希望将监控内容同步到本地 vault
- 需要灵活选择同步目标

## 🏗️ 架构设计

### 知识库工厂模式

```
KnowledgeBaseFactory
    ├── create_syncers()     # 根据配置创建同步器
    ├── sync_all(files)      # 同步到所有启用的知识库
    └── get_syncer(name)     # 获取指定同步器

KnowledgeBaseSyncer (抽象基类)
    ├── sync(file, platform)        # 单文件同步
    ├── sync_batch(files)           # 批量同步
    └── validate_config()           # 配置验证

具体实现
    ├── FeishuSyncer      # 飞书同步器
    └── ObsidianSyncer    # Obsidian 同步器
```

### Obsidian frontmatter 示例

```yaml
---
date: 2026-04-18
source: 小宇宙
tags: [播客, 转录, AI创业]
status: new
---

# 内容标题

正文内容...
```

## 🔧 配置方式

### 环境变量配置（.env）

```bash
# 飞书（可选）
FEISHU_ENABLED=true
FEISHU_WIKI_SPACE=my_library
FEISHU_ROOT_FOLDER=内容监控

# Obsidian（可选）
OBSIDIAN_ENABLED=true
OBSIDIAN_VAULT_PATH=/path/to/your/vault
OBSIDIAN_FOLDER=内容监控
OBSIDIAN_FRONTMATTER=true
```

### 配置验证

启动时自动验证配置有效性：
- Obsidian：检查 vault 路径是否存在、是否包含 `.obsidian` 目录
- 飞书：检查 lark-cli 是否可用

## 📊 Obsidian 同步特性

| 特性 | 说明 |
|------|------|
| 平台分类 | 自动按 小宇宙/YouTube/B站/Twitter 分类 |
| frontmatter | 自动添加日期、来源、标签、状态 |
| 双链支持 | 保留 Obsidian `[[双链]]` 语法 |
| 文件命名 | 支持 `日期_标题` 或 `标题_日期` 格式 |

## 🔄 向后兼容

- **默认不启用**：没有配置任何知识库时，不执行同步
- **独立启用**：可以只启用飞书、只启用 Obsidian，或同时启用
- **API 兼容**：`--no-sync` 参数仍然有效
