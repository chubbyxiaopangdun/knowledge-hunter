# 定时任务配置

## 自动执行方案

### 方案一：飞书日历任务（推荐）

由于云电脑环境限制，建议通过飞书日历创建定时提醒：

1. **打开飞书日历**
2. **创建重复事件**：每天 08:00
3. **设置提醒**：事件开始时
4. **添加任务描述**：
   ```
   执行命令：
   cd /app/data/所有对话/主对话 && python3 ./内容监控/scripts/monitor.py
   ```
5. **收到提醒后**：复制命令到云电脑执行，或通知诺诺执行

### 方案二：让诺诺（AI助手）自动执行

告诉诺诺：
> "请帮我执行内容监控系统，检查各平台更新并同步到飞书"

诺诺会帮你执行 `monitor.py` 脚本。

### 方案三：云电脑开机自启

如果云电脑支持开机自启脚本：

```bash
# 创建自启脚本
cat > /etc/init.d/content-monitor << 'EOF'
#!/bin/bash
### BEGIN INIT INFO
# Provides:          content-monitor
# Required-Start:    $local_fs $remote_fs
# Required-Stop:     $local_fs
# Default-Start:     2 3 4 5
# Default-Stop:      0 1 6
### END INIT INFO

while true; do
    # 检查时间是否为8:00
    current_hour=$(date +%H)
    current_min=$(date +%M)
    
    if [ "$current_hour" = "08" ] && [ "$current_min" = "00" ]; then
        cd /app/data/所有对话/主对话
        python3 ./内容监控/scripts/monitor.py
        sleep 60  # 避免重复执行
    fi
    
    sleep 30  # 每30秒检查一次
done
EOF

# 添加执行权限
chmod +x /etc/init.d/content-monitor

# 设置开机启动
update-rc.d content-monitor defaults
```

## 执行日志

执行日志保存在：
- `./内容监控/日志/monitor_YYYYMMDD.log` - 每日日志
- `./内容监控/日志/report_*.json` - 执行报告

## 建议的自动化流程

```
每天 08:00
    │
    ├─→ 飞书日历提醒
    │       │
    │       └─→ 您告诉诺诺："执行内容监控"
    │               │
    │               └─→ 诺诺运行 monitor.py
    │                       │
    │                       ├─→ 小宇宙RSS检查
    │                       ├─→ YouTube频道检查
    │                       ├─→ B站UP主检查
    │                       ├─→ Twitter账号检查
    │                       ├─→ Whisper转录
    │                       └─→ 飞书同步
    │
    └─→ 飞书收到通知（新文档）
```

## 手动触发示例

需要执行时，告诉诺诺：
> "请帮我运行内容监控系统，检查今天的更新并同步到飞书"

诺诺会执行完整的监控流程。
