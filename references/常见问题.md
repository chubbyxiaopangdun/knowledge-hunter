# Content Monitor 常见问题

## 安装问题

### Q1: Whisper安装失败怎么办？

**问题：** 安装openai-whisper时报错

**解决方案：**
```bash
# 方案1：使用清华源
pip install openai-whisper -i https://pypi.tuna.tsinghua.edu.cn/simple

# 方案2：先安装PyTorch
pip install torch -i https://pypi.tuna.tsinghua.edu.cn/simple
pip install openai-whisper -i https://pypi.tuna.tsinghua.edu.cn/simple

# 方案3：使用conda
conda install -c conda-forge openai-whisper
```

### Q2: ffmpeg未安装怎么办？

**解决方案：**
```bash
# Ubuntu/Debian
sudo apt-get install ffmpeg

# macOS
brew install ffmpeg

# Windows
# 下载 https://ffmpeg.org/download.html 并添加到PATH
```

### Q3: Whisper转录很慢怎么办？

**原因：** Whisper在CPU上运行较慢

**解决方案：**
1. 使用更小的模型：`tiny` > `base` > `small`
2. 有GPU的话，安装CUDA版本的PyTorch
3. 使用faster-whisper（更快但需要额外安装）

## 使用问题

### Q4: 小宇宙转录失败怎么办？

**可能原因：**
1. 音频下载失败
2. 内存不足
3. 网络问题

**解决方案：**
```bash
# 检查音频是否下载成功
ls -la ./内容监控/输出/小宇宙测试/

# 内存不足时使用分段处理
# 脚本会自动分段处理超长音频

# 网络问题，检查代理设置
export https_proxy=http://your-proxy:port
```

### Q5: YouTube字幕提取失败？

**可能原因：**
1. 视频没有字幕
2. 网络问题
3. yt-dlp版本过旧

**解决方案：**
```bash
# 更新yt-dlp
pip install --upgrade yt-dlp

# 检查视频是否有字幕
yt-dlp --list-subs "视频URL"

# 无字幕时自动使用Whisper转录
```

### Q6: B站视频下载失败？

**可能原因：**
1. 视频被删除
2. 需要登录态（HTTP 412错误）
3. API限流
4. 地区限制

**解决方案：**

**方案1：配置登录Cookie（推荐）**
```bash
# 1. 从浏览器获取SESSDATA
# 登录 bilibili.com → F12 → Application → Cookies → bilibili.com → 复制SESSDATA值

# 2. 设置环境变量
export BILIBILI_SESSDATA="你的SESSDATA值"

# 3. 运行脚本
python3 ./内容监控/scripts/bilibili.py
```

**方案2：等待几分钟后重试**

**方案3：检查UP主主页确认视频是否存在**

**方案4：使用you-get备用下载工具**

**注意：** B站大部分中字视频是硬字幕（内嵌），API无法提取，需使用Whisper转录。

### Q7: Twitter推文抓取失败？

**可能原因：**
1. 反爬虫机制
2. 账号被限制

**解决方案：**
1. 降低抓取频率
2. 使用官方API（付费）
3. 手动复制推文内容

## 飞书同步问题

### Q8: 飞书授权失败？

**解决方案：**
```bash
# 重新授权
lark-cli auth login --domain drive,wiki,docs

# 检查授权状态
lark-cli auth status
```

### Q9: 同步到飞书失败？

**可能原因：**
1. 授权过期
2. 网络问题
3. 文件过大

**解决方案：**
1. 重新授权飞书
2. 检查网络连接
3. 大文件分批上传

## 转录质量问题

### Q10: 转录质量不好怎么办？

**解决方案：**
1. 使用更大的模型
   ```python
   # 从 tiny -> base -> small -> medium
   model = whisper.load_model('small')
   ```

2. 指定语言
   ```python
   result = model.transcribe(audio, language='zh')
   ```

3. 使用有字幕的视频源

---

### Q11: 时间轴不准确？

**原因：** Whisper的时间轴是估算的

**解决方案：**
1. 对于YouTube/B站，优先使用官方字幕
2. 使用更大的模型可以提高时间轴准确度

## 配置问题

### Q12: 如何添加新的监控博主？

**编辑配置文件：**
```bash
vim ./内容监控/配置/博主列表.md
```

**添加格式：**
```markdown
| 名称 | RSS地址/频道ID/UID | 备注 |
|------|-------------------|------|
| 新博主 | 链接 | 说明 |
```

### Q13: 如何暂停某个博主的监控？

**方法：** 在配置文件中注释该行
```markdown
# | 已暂停的博主 | 链接 | 备注 |
```

### Q14: 如何设置每日监控时间？

**创建定时任务：**
告诉诺诺"创建每日内容监控任务，每天8点执行"

或手动创建calendar任务

## 性能问题

### Q15: 占用太多存储空间？

**解决方案：**
```bash
# 清理临时音频文件
rm -rf ./内容监控/输出/*/audio_segments/

# 只保留最近30天的音频
find ./内容监控/输出 -name "*.m4a" -mtime +30 -delete
```

### Q16: 转录时内存不足？

**解决方案：**
1. 脚本已自动分段处理
2. 使用更小的模型
3. 关闭其他程序释放内存

## 其他问题

### Q19: 支持哪些音频/视频格式？

**支持格式：**
- 音频：mp3, m4a, wav, flac, ogg
- 视频：mp4, mkv, avi, mov, webm

### Q20: 转录结果可以导出吗？

**可以！**
- Markdown格式：直接下载.md文件
- 纯文本：去掉时间轴即可
- Word：使用pandoc转换
  ```bash
  pandoc transcript.md -o transcript.docx
  ```
