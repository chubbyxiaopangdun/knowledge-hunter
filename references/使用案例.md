# Content Monitor 使用案例详解

## 案例1：小宇宙播客转录

### 场景描述
用户想要将一期小宇宙播客转录成文字稿，便于阅读和检索。

### 操作步骤

**方式一：直接请求**
```
用户：帮我把这个小宇宙节目转成文字稿
      https://www.xiaoyuzhoufm.com/episode/69cfdd21e2c8be31555303e4

诺诺：
1. 获取节目信息
2. 下载音频（114MB，123分钟）
3. 使用Whisper base模型转录
4. 生成带时间轴的文字稿
5. 同步到飞书知识库

✅ 转录完成！
📄 文件：给女孩的商业第一课_AI时代30分差生.md
📊 5056行，170KB
```

**方式二：批量转录**
```
用户：帮我转录纵横四海最近的3期节目

诺诺：
正在获取纵横四海播客信息...
找到RSS源，正在解析最新节目...

✅ 已添加到转录队列：
- EP89: 《原子习惯》精读
- EP88: 《刻意练习》解读  
- EP87: 《费曼学习法》分享

预计转录时间：45分钟
完成后会通知您并同步到飞书
```

### 输出示例

## 总结

这些案例展示了Content Monitor技能的核心使用场景：

1. **单次转录**：快速获取内容文字稿
2. **持续监控**：自动追踪更新
3. **知识管理**：系统性收集内容
4. **观点提取**：提炼核心价值
5. **批量管理**：高效运维监控列表

通过灵活组合这些功能，可以满足各种内容收集和学习需求。
