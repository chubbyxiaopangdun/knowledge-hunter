# 知识猎手 (Knowledge Hunter) 🎯

> 主动出击，精准捕获有价值的内容，转化为你的知识资产

**版本**: v1.5.0 | **状态**: 正式版

---

## 一句话介绍

**从「内容消费」到「内容创作辅助」** —— 让AI帮你监控、转录、发现热点、生成选题

---

## ✨ 核心功能

### 📥 多平台监控（v1.0）

| 平台 | 功能 | 特点 |
|------|------|------|
| 🎙️ 小宇宙 | 播客监控 | RSS订阅，自动获取音频 |
| 📺 YouTube | 视频监控 | 支持多频道，字幕提取 |
| 📺 B站 | 视频监控 | 支持登录态（可选），CC字幕 |
| 🐦 Twitter | 推文监控 | 公开推文抓取 |

### 📝 智能转录（v1.0）

- ✅ **完全免费**：使用Whisper开源模型本地运行
- ✅ **高准确率**：支持多语言，保留时间轴
- ✅ **观点提取**：自动提取核心观点和金句

### 📰 每日简报（v1.1）

每天早上9点自动推送：
- 🎬 3个值得看的视频
- 🎧 2个值得听的播客  
- 💡 1个观点冲突

### 🔥 选题发现器（v1.2）⭐ 核心升级

解决创作者痛点：**知道要做内容，但不知道写什么**

| 功能 | 说明 |
|------|------|
| 🔥 **热点发现** | 自动分析本周内容，发现热点话题 |
| ⚖️ **观点对比** | 发现不同观点的碰撞，适合做对比内容 |
| 📌 **选题追踪** | 追踪你的选题相关的新内容 |
| ✏️ **选题生成** | 基于热点生成具体选题建议 |
| 📋 **周报推送** | 周日自动生成选题报告 |

### 📚 多知识库同步（v1.3）⭐ 最新升级

支持同时同步到多个知识库，用户按需选择：

| 知识库 | 同步方式 | 配置难度 |
|--------|---------|---------|
| **飞书** | 通过 lark-cli 上传到知识库 | ⭐⭐ 需安装 lark-cli |
| **Obsidian** | 直接写入本地 vault 文件夹 | ⭐ 仅需指定路径 |

Obsidian 特性：
- 自动添加 YAML frontmatter（日期、来源、标签、状态）
- 支持 `[[]]` 双链语法
- 按平台自动分文件夹

### ✨ 金句提取器（v1.4）⭐ 最新升级

从「监控工具」升级为「素材工具」，自动从转录内容中提取金句，存入素材库供内容创作使用。

| 功能 | 说明 |
|------|------|
| 🔥 **AI金句提取** | 自动从转录文本中提取约20个金句 |
| 🏷️ **自动打标签** | AI自动识别主题（创业/职场/AI等） |
| 📚 **素材库管理** | 按主题分类，支持搜索和导出 |
| 💡 **创作素材** | 做内容时可从素材库快速提取金句 |

**使用流程**：
```
内容 → 转录 → AI自动提取20个金句 → 进入「我的素材库」 → 用户做内容时提取金句
```

### 🔄 全网资源对比（v1.5）⭐ 最新升级

解决痛点：**平台限制导致转录卡住**

| 功能 | 说明 |
|------|------|
| 🔍 **智能检测** | 自动识别平台限制（403/404/私密/下架） |
| 🌐 **全网搜索** | 在 YouTube/B站/Apple Podcasts 搜索替代资源 |
| ✅ **可用性验证** | 验证找到的资源是否可转录 |
| 🔄 **自动切换** | 使用替代资源继续转录，记录来源变更 |

**工作流程**：
```
转录失败 → 检测是否平台限制 → 全网搜索替代资源 → 验证可用性 → 自动切换转录
```

### 🎯 每日创意燃料（v2.0）⭐⭐⭐ 规划中

> ⚠️ **此功能正在开发中，预计在 v2.0 版本发布**

从「选题发现」升级为「创意燃料」，**每天早上告诉你：今天做什么内容**。

| 功能 | 说明 |
|------|------|
| 🔥 **今日热点** | 从过去24小时内容中提取3个可追热点 |
| 💡 **今日金句** | 提取5个可直接引用的素材 |
| ✏️ **今日可做** | 生成1个可直接发布的Thread/小红书文案 |
| 📊 **数据看板** | 展示今日数据统计和趋势 |

**一句话定位**：让用户「看完就能动手」，不是「看完存起来」

```
📋 今日创意燃料 | 2026-04-18 周六

🔥 今日热点（3个可追热点）
1️⃣ AI Agent创业 - 热度85/100
   💡 建议选题：「AI Agent创业的3个陷阱」

💡 今日金句（5个可直接引用的素材）
「AI不会取代你，但会用AI的人会取代你」— 某CEO

✏️ 今日可做（1个可直接发布的Thread）
【Tweet 1/5】{钩子}
...
```

**与之前版本的关系**：
- `选题报告`（v1.2）= 周维度，给你看这周有什么
- `每日创意燃料`（v2.0）= 日维度，告诉你今天做什么

两个功能可以并存，互为补充。

## 🚀 快速开始

### 安装依赖

```bash
# 克隆仓库
git clone https://github.com/chubbyxiaopangdun/knowledge-hunter.git
cd knowledge-hunter

# 安装依赖
pip install -r scripts/requirements.txt

# 安装FFmpeg（转录必需）
# Ubuntu/Debian: sudo apt install ffmpeg
# macOS: brew install ffmpeg
# Windows: 从 https://ffmpeg.org/download.html 下载
```

### 配置监控列表

编辑 `templates/博主列表.md`，添加你要监控的博主：

```markdown
## 小宇宙播客
| 名称 | RSS地址 |
|------|---------|
| 示例播客 | https://feed.xyzfm.space/xxx |

## YouTube频道
| 名称 | 频道ID |
|------|--------|
| 示例频道 | UCxxxxxxx |

## B站UP主
| 名称 | UID |
|------|-----|
| 示例UP主 | 12345678 |
```

### 配置知识库同步（可选）

```bash
# 复制配置模板
cp .env.example .env

# 编辑 .env 启用知识库
# 飞书
FEISHU_ENABLED=true
FEISHU_WIKI_SPACE=my_library

# Obsidian
OBSIDIAN_ENABLED=true
OBSIDIAN_VAULT_PATH=/path/to/your/vault
```

### 运行监控

```bash
# 执行一次监控
python scripts/monitor.py

# 生成选题报告
python scripts/topic_report.py

# 生成每日简报
python scripts/daily_briefing.py
```

```bash

```

### ⚠️ 配置迁移（重要）

v1.3.0 对知识库同步进行了架构升级：

```bash
# 1. 创建 .env 文件
cp .env.example .env

# 2. 启用需要的知识库
# 编辑 .env 文件：
FEISHU_ENABLED=true          # 启用飞书
OBSIDIAN_ENABLED=true        # 启用 Obsidian
OBSIDIAN_VAULT_PATH=/path    # Obsidian vault 路径
```

- ✅ 默认不启用任何知识库同步
- ✅ 监控、转录、观点提取等功能不受影响
- ✅ 原有配置方式仍然支持

## 💡 使用场景

### 场景1：日常内容监控

```bash
# 每天自动监控更新
python scripts/monitor.py

# 查看转录结果
ls 输出/B站/
ls 输出/YouTube/
```

### 场景2：生成周选题报告

```bash
python scripts/topic_report.py

# 输出示例
📋 本周选题发现 | 2026-W16

🔥 热点话题
1️⃣ AI创业 - 热度 +180%
2️⃣ 私域2.0 - 热度 +120%

✏️ 建议选题
8个选题建议已生成
```

### 场景3：同步到 Obsidian

```bash
# 配置 .env
OBSIDIAN_ENABLED=true
OBSIDIAN_VAULT_PATH=/Users/xxx/Documents/ObsidianVault

# 运行监控，自动同步
python scripts/monitor.py

# 在 Obsidian 中查看
# 内容监控/小宇宙/2026-04-18_xxx.md
```

### 场景4：配置定时任务

```bash
# 每天早上9点生成简报
# 每周日晚上8点生成选题报告

# 通过cron设置
0 9 * * * python scripts/daily_briefing.py
0 20 * * 0 python scripts/topic_report.py
```
