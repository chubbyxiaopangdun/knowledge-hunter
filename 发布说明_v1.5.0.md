## 🎉 新功能：全网资源对比

当小宇宙等平台限制音频转录时，自动寻找可转录的替代资源，再也不怕「资源被吞」。

### 解决的问题

- ❌ 小宇宙音频链接失效/被限制
- ❌ 下载时遇到 403/404 错误
- ❌ 内容已下架或设为私密
- ❌ 手动找替代资源费时费力

### 工作流程

```
转录失败 
  → 检测是否平台限制（403/404/私密/下架等）
  → 提取内容标题/作者
  → 全网搜索替代资源（YouTube → B站 → Apple Podcasts）
  → 验证资源可用性
  → 选择最佳资源
  → 重新转录
  → 记录资源来源变更
```

### 支持的替代平台

| 平台 | 优先级 | 转录支持 |
|------|--------|---------|
| YouTube | ⭐⭐⭐ (最高) | ✅ |
| B站 | ⭐⭐ | ✅ |
| Apple Podcasts | ⭐ | ✅ |
| Spotify | ⭐ | ✅ |

## ⚙️ 配置说明

### 新增配置项（可选）

在 `.env` 文件中添加以下配置（可选，有默认值）：

```bash
# 全网资源对比配置
ENABLE_FALLBACK_SEARCH=true          # 是否启用（默认开启）
FALLBACK_PLATFORMS=youtube,bilibili,apple_podcasts  # 搜索平台
FALLBACK_MAX_RESULTS=10              # 最大搜索结果数
FALLBACK_TIMEOUT=300                 # 超时时间（秒）
```

### 日志文件

| 文件 | 说明 |
|------|------|
| `日志/fallback_search.log` | 搜索事件日志 |
| `日志/fallback_attempts.jsonl` | Fallback尝试记录 |
| `日志/source_changes.jsonl` | 资源来源变更记录 |

## 🚀 使用方式

### 命令行搜索替代资源

```bash
# 基本搜索
python scripts/fallback_handler.py --title "播客标题" --author "播客名"

# 验证资源可用性
python scripts/fallback_handler.py --title "标题" --verify

# 指定内容类型
python scripts/fallback_handler.py --title "标题" --type video
```

### 自动处理

当监控任务遇到平台限制时，会自动：
1. 检测是否为平台限制错误
2. 搜索替代资源
3. 验证可用性
4. 选择最佳资源
5. 完成转录

转录结果会自动标注来源变更（如有）。
